# Plan: QMD + Obsidian Second Brain Installation Script

## Context

Harsha wants a unified "second brain" system combining QMD (local semantic search engine) + Obsidian vault. This replaces the cloud-based Supabase approach from doc 03 with the simpler, local-only architecture from doc 05. ## What the Script Does

A single interactive `install.sh` that:

1. **Detects platform** (Linux/WSL/macOS) and adjusts install commands
2. **Asks for input**:
   - Vault path (default: `~/vault`)
   - Which collection groups to enable: "dev harness" (notes, daily, sessions, transcripts, skills) + "second brain" (thoughts, people, references, weekly-reviews)
   - Whether to set up the `/recall` Claude Code skill
   - Whether to set up the session sync script + Claude Code hook
   - Whether to configure QMD MCP server for Claude Desktop
3. **Installs prerequisites**: Bun runtime, QMD (via `bun install -g`)
4. **Creates vault folder structure** with all selected collection directories
5. **Creates template files** (daily template, weekly review template) inside the vault
6. **Registers QMD collections** with context descriptions
7. **Runs `qmd embed`** to build the initial index
8. **Optionally sets up**:
   - `/recall` skill at `~/.claude/skills/recall/SKILL.md`
   - Session sync script at `~/.local/bin/sync-claude-sessions.sh`
   - Claude Code hooks config for auto-sync on session close
   - QMD MCP server config for Claude Desktop

## Key Design Decisions

- **Single script, no external dependencies** — pure bash, works immediately
- **Idempotent** — safe to re-run (uses `mkdir -p`, collection add with `|| true`, etc.)
- **Non-destructive** — never overwrites existing files, checks before writing
- **Backs up** existing configs (hooks.json) before modifying
- **WSL-aware** — detects WSL for Claude Desktop config path differences
